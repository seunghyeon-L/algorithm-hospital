1. 실제 의료 현장의 수술 스케줄링

병원은 수학적 최적화보다 "운영 규칙 + 데이터 기반 관리"로 OR(수술실) 일정을 운영합니다.

🗓️ 배정 방식: 블록 vs 오픈

블록 스케줄링(Block): 과·외과의·그룹에게 정해진 OR 시간블록을 미리 배정. 대부분의 병원이 채택. 예측 가능하고 외과의 만족도가 높지만, 블록이 비면 낭비(그래서 미사용 블록을 release해 재활용).

오픈 스케줄링(Open): 먼저 요청하는 순서로 배정. 유연하지만 혼잡·예측 어려움.

📎 출처: Hospital Medical Director — OR Block vs Open Scheduling

📊 데이터 기반 운영

블록 최적화는 첫 케이스 시작시각·케이스 소요시간·블록 종료·취소율·블록 release 비율·전환시간 같은 실측 데이터를 EMR(전자의무기록)과 연동해 관리합니다. 유사 시술을 묶으면 전환시간↓·가동률↑.

2. 스케줄링 알고리즘 landscape

학계는 수술실 스케줄링(ORS)을 의사결정 수준과 해법 범주로 분류합니다.

🧭 의사결정 수준

수준무엇을 정하나예
전략(Strategic)장기 case mix·OR 용량 계획과별 OR 시간 총량 배분
전술(Tactical)주기적 마스터 수술 일정(MSS)요일별 블록 배정 (Beliën 2009)
운영(Operational)당일 케이스 배정·순서·방 지정case sequencing (Cardoen 2009) ← 이 프로젝트

⚙️ 해법(알고리즘) 범주 — 각각 무엇이고 어떤 트레이드오프인가

정확해 정수계획(MILP) · 제약프로그래밍(CP-SAT) 이 프로젝트 채택 (RCPSP)

문제를 변수·제약·목적식의 수식으로 표현하고 솔버가 수학적 최적(또는 증명된 근접해)을 찾는다. 리뷰에 따르면 ORS에서 가장 널리 쓰이는 모델이며 현장 주류.

✓ 장점: 최적/최적성 보증, 제약 표현이 명확·유연

✗ 단점: 대규모에서 계산시간 급증, 모델링 난도 높음

적용: 우리 rcpsp.py가 OR-Tools CP-SAT로 자원제약 스케줄링을 푼다.

메타휴리스틱 유전 알고리즘(GA)·시뮬레이티드 어닐링(SA)·타부서치·ACO·PSO 이 프로젝트 채택 (GA · SA)

최적성은 보장하지 않지만 "좋은 해"를 빠르게 탐색. 큰 문제·복잡한 목적에 강하다. ORS에서 GA·Tabu·SA·하이브리드(GA+Tabu, Fei 2006/2010)·ACO(Xiang 2017)·PSO(Wang 2014)가 널리 연구됨.

✓ 장점: 확장성·속도, 복잡/비선형 목적 처리 용이

✗ 단점: 최적성 미보장, 파라미터 튜닝 필요, 재현성 관리 필요

적용: ga.py(GA, 군집 기반·DEAP)와 sa.py(SA, 궤적 기반·담금질) 두 메타휴리스틱을 구현. 둘 다 적합도=Σ대기, 동일 디코더·시드 고정.

다목적 NSGA-II 등 (Pareto 최적)

대기시간·makespan·가동률·공정성처럼 상충하는 여러 목표를 동시에 최적화해 Pareto 해집합을 제시. OR 스케줄링용 개선 NSGA-II(2019)·다목적 VNS(2021) 연구가 있음.

✓ 장점: 의사결정자가 트레이드오프를 보고 선택 가능

✗ 단점: 구현·해석 복잡, 단일 "정답"이 아님

우리 프로젝트: 현재 단일목적(Σ대기). 다목적은 향후 과제(→ 5절).

학습 기반 머신러닝 · 강화학습(RL)

ML은 케이스 소요시간 예측(Bartek 2019)에, 강화학습은 불확실성 하 동적 재스케줄링에 쓰인다. 최근 다중에이전트 RL로 당일(intraday) OR 스케줄링을 다룬 연구(2025)가 등장.

✓ 장점: 불확실성·실시간 적응에 강함

✗ 단점: 데이터·학습 비용, 해석성·검증 난도 높음

우리 프로젝트: 범위 밖(교육용 비교에 과함). 설명서에 동향으로만 소개.

그래프·시뮬레이션 위상정렬·임계경로(CPM)·이산사건 시뮬레이션 이 프로젝트 채택 (교안)

✓ 장점: 이해·구현 쉬움, 문제 구조를 드러냄

✗ 단점: 그 자체로는 자원 최적화를 못함(분석 도구)

옷 입는 순서를 생각하세요. 양말을 신어야 신발을 신고, 속옷을 입어야 바지를 입습니다. 이런 "먼저 해야 하는 것"을 모두 지키면서 전체 순서를 정하는 게 위상정렬입니다.

🏥 우리 병원 문제에선

환자는 보통 [검사 → 마취 준비 → 수술 → 회복] 순서를 따릅니다. 검사가 안 끝났는데 수술을 시작할 순 없죠. 위상정렬이 이 모든 선후 제약을 지키는 실행 순서를 만들어줍니다.

⚙️ 코드로는 어떻게

graph.py · topological_order() — 들어오는 화살표(선행 작업)가 없는 작업부터 꺼내고, 그 작업을 지우면 새로 선행이 사라진 작업을 또 꺼내는 식으로 반복합니다. NetworkX의 lexicographical_topological_sort를 써서 같은 입력엔 항상 같은 순서가 나오게(재현 가능) 했습니다.

👁️ 화면·결과에선

이 순서가 베이스라인·GA가 일정을 짜기 시작하는 "우선순위 명단"이 됩니다. 웹의 DAG 그래프 탭에서 작업들이 왼쪽→오른쪽 단계로 늘어선 모습이 바로 위상순서의 시각화입니다.

작동 원리 (단계)
# 각 작업의 진입차수(선행 개수)를 세고, 0인 것부터 처리 for v in 작업들: indeg[v] = (v의 선행 개수) 큐 Q ← indeg가 0인 작업들 # 선행이 없는 = 바로 시작 가능 결과 ← [] while Q 비어있지 않음: u ← Q에서 꺼냄 # (결정적: 이름이 가장 작은 것) 결과.append(u) for u의 후속 w: indeg[w] -= 1 if indeg[w] == 0: Q에 추가 if len(결과) < 작업 수: 순환 존재(불가능)
수치 워크스루 — 4작업 예시

작업 A·B·C·D, 간선: A→B, A→C, B→D, C→D. 진입차수: A=0, B=1, C=1, D=2.

# 초기: Q = {A} (indeg=0) step 1: 꺼냄 A → 결과=[A], B.indeg=0→Q, C.indeg=0→Q Q = {B, C} step 2: 꺼냄 B (사전순 먼저) → 결과=[A,B], D.indeg=1→아직 대기 Q = {C} step 3: 꺼냄 C → 결과=[A,B,C], D.indeg=0→Q Q = {D} step 4: 꺼냄 D → 결과=[A,B,C,D] ✓ 완료: 4=4작업, 순환 없음

D는 B·C 양쪽이 끝나야 꺼낼 수 있으므로 마지막. B·C 순서는 사전순으로 결정(재현성).

👁️ 화면·결과에선

요약 카드의 "임계경로 길이(이론적 하한)", 그리고 DAG 탭에서 주황색으로 강조된 경로가 그것입니다. 네 알고리즘(베이스라인·RCPSP·GA·SA)의 완료시각(makespan)을 이 하한과 비교해 "최적에 얼마나 가까운지" 가늠합니다.

작동 원리 (forward pass)
# 위상순서대로 ES(가장 이른 시작)·EF(가장 이른 종료)를 전진 계산 for v in 위상순서: ES[v] = max( EF[u] for u in v의 선행 ) # 선행 없으면 0 EF[v] = ES[v] + duration[v] 임계경로 길이 = max(EF[v]) 임계경로 = EF가 최대인 끝점에서 선행을 역추적
핵심 수식 — ES(v) = maxu→v EF(u),   EF(v) = ES(v) + d(v)
여유(float/slack) = LS − ES, 임계경로 위 작업은 float = 0(지연되면 전체가 늦어짐).
수치 워크스루 — forward pass 예시

작업 A(d=10), B(d=20), C(d=40), D(d=5). 간선: A→B, A→C, B→D, C→D.

# 위상순서: A → B → C → D ES[A]=0, EF[A]=0+10=10 ES[B]=EF[A]=10, EF[B]=10+20=30 ES[C]=EF[A]=10, EF[C]=10+40=50 ES[D]=max(EF[B],EF[C])=max(30,50)=50, EF[D]=50+5=55 ← 임계경로 길이 여유(float): B의 LS=50-5-20=25 → float=25-10=15 (15분 지연 허용) C의 float=0 → 임계작업 (C가 1분 늦으면 전체가 1분 늦어짐)

임계경로: A→C→D (합=55). B는 여유가 있으므로 임계 아님.

CP-SAT는 내부적으로 어떻게 푸나

제약전파(CP) + SAT(CDCL) + 선형완화(LP)를 포트폴리오로 병행합니다. ① 제약전파: cumulative·범위 추론으로 변수 도메인을 좁힘. ② Lazy Clause Generation: 전파기가 "왜 도메인을 줄였는지"를 절(clause)로 SAT 코어에 학습 → 비슷한 실패를 즉시 가지치기. ③ CDCL: 충돌을 분석해 nogood 절을 만들고 backjump. → worst-case는 지수적이지만 실무에선 학습·전파로 빠르게 수렴.

교차 연산자 비교: 순열 인코딩에서 세 교차 방식의 특성을 비교하면 — OX(Order Crossover, Davis 1985)는 선행 순서를 보존해 스케줄링처럼 순서가 중요한 문제에 적합하다. PMX(Partially Mapped Crossover, Goldberg & Lingle 1985)는 위치 매핑을 보존해 TSP에 좋지만, 순서보다 절대위치가 중요한 경우에 유리하다. CX(Cycle Crossover, Oliver 1987)는 각 유전자의 절대 위치를 최대한 유지하며, 특정 문제에선 OX보다 나쁜 해를 내기도 한다. 스케줄링 우선순위 순열에선 순서 정보가 핵심이므로 OX가 표준 선택이다(GeneticOperations.pdf).

선택 연산자 비교: 토너먼트 선택(우리 구현)은 구현이 단순하고 선택 압력을 k로 직접 조절할 수 있어 스케줄링에 널리 쓰인다. 룰렛 선택(fitness-proportionate)은 적합도 차이가 클 때 조기 수렴 위험이 있고, 부동소수점 스케일에 민감하다. 순위 선택(rank selection)은 적합도 절대값이 아닌 순위를 사용해 조기 수렴을 억제하지만, 세대 초반 수렴이 느릴 수 있다.

엘리트주의(Elitism): Hall-of-Fame(HOF)으로 최고 개체를 다음 세대에 보존하면 수렴 보장이 강화된다. HOF=1이 우리 구현의 기본값.

👁️ 화면·결과에선

GA(군집 기반)와 대비되는 궤적 기반 메타휴리스틱. 2D 동선·간트에서 SA를 골라 GA·RCPSP와 배치를 비교할 수 있습니다. 같은 시드면 결과 재현. 메타휴리스틱 내부 비교(GA vs SA)까지 가능합니다.

현재 ← 위상순서(초기해); 최고 ← 현재 T ← 초기온도 while 반복 < max_iters (또는 시간예산): 이웃 ← 현재에서 두 위치를 교환 Δ = cost(이웃) − cost(현재) # cost = Σwait if Δ ≤ 0 or random() < exp(−Δ / T): # Metropolis 현재 ← 이웃 if cost(현재) < cost(최고): 최고 ← 현재 T ← max(T_end, α · T) # 기하적 냉각 return 최고 → 디코더 → 일정
수용 확률(Metropolis)   P(수용) = min{ 1, exp(−Δ / T) }   (Boltzmann 분포에서 유래)
T가 크면 나쁜 해도 잘 받아들임(탐험) → 냉각될수록 좋은 해만(수렴).
수치 워크스루 — 3스텝 예시

초기 Σwait=200, T₀=30(≈200×0.15), α=0.9975.

# iter 1: 이웃 swap → cost=195, Δ=−5 (개선) Δ=-5 ≤ 0 → 무조건 수용. 현재=195, 최고=195. T=30×0.9975=29.93 # iter 2: 이웃 swap → cost=198, Δ=+3 (악화) P = exp(−3/29.93) = exp(−0.100) ≈ 0.905 random()=0.72 < 0.905 → 수용(탈출 시도). 현재=198, 최고=195. T=29.86 # iter 3: 이웃 swap → cost=190, Δ=−8 (개선) Δ=-8 ≤ 0 → 무조건 수용. 현재=190, 최고=190. T=29.79 # ↑ iter2에서 나쁜 해를 받아들이지 않았으면 190을 탐색하지 못했을 수도 있음

초기온도가 높을수록 악화 수용 확률이 높아(탐험) 지역 최적 탈출 가능성↑. T가 식을수록 수용 확률→0(수렴).

파라미터 민감도 (Parameter Sensitivity)
파라미터의미우리값민감도
초기온도 T₀초기 탐험 강도초기비용 × 0.15★★★ 가장 중요. 너무 낮으면(≈0) 첫 스텝부터 개선만 수용 → 그리디와 동일. 너무 높으면 무작위 탐색. 문헌 기준: 초기 악화 수용률 ≈80%가 되도록 T₀를 설정하는 것이 표준(Kirkpatrick 1983; Aarts & van Laarhoven 1985). 우리 구현의 ×0.15는 이 경험칙을 따름.
냉각률 α기하적 냉각 비율0.9975★★★ 1에 가까울수록 천천히 식어 탐색 시간↑·해 품질↑(but 느림). 0.99 vs 0.9975는 동일 반복 수에서 최종 온도가 약 5배 차이. α=0.99면 6000번 후 T≈T₀×0.99⁶⁰⁰⁰≈ T₀×9×10⁻²⁷ (완전 냉각); 0.9975면 T≈T₀×0.22 (아직 탐험 여지). max_iters와 반드시 함께 조정.

초기해탐색 시작점위상순서(=baseline)★ 좋은 초기해에서 시작할수록 수렴 빠름. 우리 구현은 위상순서(=baseline 해)를 출발점으로 써서 탐색 시작 품질을 높임.
대표 변형(Variants)

재가열(Reheating): 일정 반복마다 또는 개선이 없을 때 T를 다시 높이는 기법. 냉각 후 지역 최적에 빠진 경우 다시 탐험하게 해 주지만, 수렴 보장이 약해진다. 구현: 일정 iter 동안 최고해 갱신이 없으면 T를 T₀ × 0.5로 재설정.

적응형 냉각(Adaptive Cooling): 현재 수용률을 실시간으로 측정해 목표 수용률(예: 초반 80% → 말기 1%)을 유지하도록 T를 동적으로 조정. 표준 기하적 냉각보다 파라미터 민감도가 줄어든다(Adaptive temperature control, ResearchGate; arXiv 2002.06124).

List-based SA (LBSA): 단일 T 대신 최근 비용 변화 목록을 유지해 냉각을 결정하는 변형으로, 초기온도 설정 문제를 줄인다(PMC4808530).

시간복잡도 ≈ O(반복수 × 디코딩비용) · 출처: Kirkpatrick, Gelatt & Vecchi, “Optimization by Simulated Annealing”, Science, 1983 · Metropolis et al. 1953

⚖️ 핵심 트레이드오프 — 세 가지 긴장

① 정확해(CP-SAT) vs 메타휴리스틱(GA·SA)

"증명된 최적" 한 축과 "빠른 좋은 해" 한 축의 맞교환.

CP-SAT (RCPSP)

충분한 시간이 있으면 최적 또는 증명된 근접해를 낸다. 제약·목적을 수식으로 정확히 표현. 단점: 작업 수가 커지거나(NP-hard) 시간예산이 짧으면 미수렴하여 best-found가 나쁠 수 있고, 모델링 난도가 높다.

GA · SA (메타휴리스틱)

최적성은 보장하지 못하지만 어떤 규모든 그 시간 안의 "좋은 해"를 빠르게 낸다. 목적이 복잡·비선형이어도 잘 동작. 단점: 보장이 없어 결과를 "증명"할 수 없고, 파라미터 튜닝이 필요하다.

📊 우리 실측 (n=20, seed 7, 8초 예산): CP-SAT가 8초 안에 최적에 못 닿아 GA·SA(−52.7%)가 CP-SAT(−45.2%)보다 좋았음. 하지만 시간을 늘리면 CP-SAT가 최적/근접으로 역전한다 → 시간예산이 이 트레이드오프의 핵심 변수. 즉 "정확해가 항상 이긴다"가 아니라 "시간이 충분하면 이긴다".

② 군집(GA) vs 궤적(SA) — 메타휴리스틱 내부의 선택

여러 해를 동시에 진화시키나, 한 해를 빠르게 옮기나.

GA — 군집 기반

교차(OX)로 서로 다른 해의 정보를 결합해 전역 탐색·다양성에 강하다. 다봉성(여러 골짜기) 문제에 유리. 단점: 세대마다 개체수만큼 평가(무겁다), 조정할 파라미터가 많다(pop·세대·교차·돌연변이·토너먼트).

SA — 궤적 기반

한 번에 한 해만 다루고 스텝당 평가 1회로 가볍고 빠르다. 파라미터가 적다(온도·냉각률). 단점: 교차가 없어 멀리 떨어진 좋은 해를 "결합"하는 힘은 약하다.

📊 우리 실측: SA가 GA와 동일 품질(−52.7%)을 1/3 시간(2.58s vs 7.57s)에 도달. 이 인스턴스에선 SA가 효율적이지만, 문제가 더 크고 골짜기가 많아질수록 GA의 다양성이 유리해질 수 있다 → 정답은 "둘 다 돌려 비교".

③ 최적화 vs 분석 — 베이스라인·그래프분석의 역할

베이스라인·위상정렬·CPM은 경쟁자가 아니라 토대이자 잣대다.

• 베이스라인(그리디) = "단순 순서"의 결과 → 개선율 0%의 기준선. 이게 없으면 "N% 단축"을 말할 수 없다.
• CPM 임계경로 = 자원 무시 도달 불가 하한 → 최적화 결과가 "이론 한계에 얼마나 가까운지" 가늠.
• 위상정렬 = 세 최적화기 모두의 출발 순서(baseline·GA·SA 디코더, RCPSP 선후제약).

🧭 상황별 선택 가이드
• 중소규모 + 정확성·검증 중요 + 시간 여유 → RCPSP (CP-SAT)
• 대규모 + 빠른 "좋은 해"가 목표 → GA 또는 SA
• 빠른 단일해 + 튜닝 최소 → SA
• 다목적(대기 + 가동률 + 공정성) 동시 → NSGA-II (향후)
• 불확실·실시간 재계획 → 강화학습 (향후)
• 즉답 기준선·디버깅 → 베이스라인 그리디

→ 이 프로젝트가 네 알고리즘을 모두 비교하는 이유: 정확해(CP-SAT) + 두 계열 메타휴리스틱(군집 GA · 궤적 SA) + 기준선을 한 무대에 올려, 위 트레이드오프를 주장이 아니라 실측 데이터로 보여주기 위함입니다.

4. 이 프로젝트의 선택과 근거

선택: 교안 그래프 분석(위상정렬·CPM·SCC) + RCPSP(CP-SAT) + GA(DEAP) + SA(담금질)의 4자 비교.
선택왜
위상정렬·CPM·SCC과목이 그래프 알고리즘 기반을 요구. 선후관계 모델링·임계경로 하한·사이클 검증의 토대.
RCPSP via CP-SAT리뷰가 지목한 현장 주류(정수/제약계획). 자원제약·목적식을 정확히 표현하고 최적/근접해를 보장 → 비교의 "정답" 기준.
GA(군집 메타휴리스틱)현장·논문에서 가장 널리 연구된 메타휴리스틱. 여러 후보를 진화시키는 군집 기반. 확장성·속도 축을 대표.
SA(궤적 메타휴리스틱)단일 해를 이웃해로 옮기며 온도에 따라 나쁜 해도 수용해 지역 최적 탈출. GA(군집)와 대비되는 계열을 더해 메타휴리스틱 내부 비교까지 가능.
RL/다목적 제외교육용 비교에는 과함. 설명서에 동향으로 소개하고 향후 과제로 남김.

→ 즉, 정확해(CP-SAT) + 두 계열의 메타휴리스틱(군집 GA · 궤적 SA)을 교안 그래프 분석 위에 얹어 공정 비교한 것이 이 프로젝트의 설계입니다.
