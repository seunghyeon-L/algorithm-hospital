🏥 병원 수술 스케줄링 — 개념 적용 맵

기획 단계의 개념들이 실제로 무엇이고, 어떻게 코드로 구현됐으며, 화면에서 어떻게 표현되는지를 처음 보는 사람도 이해할 수 있게 풀어 설명합니다.

현실 문제 위상정렬 임계경로·CPM SCC RCPSP 유전 알고리즘 시뮬레이티드 어닐링 베이스라인 공정 비교 도구·데이터 확장요소 결과

풀어야 할 현실 문제

기획 때 정의한 두 가지 핵심 질문. 이 프로젝트의 모든 개념은 결국 이 둘을 풀기 위한 도구입니다.

Q1. 수술·검사 30~50건에 선후관계가 있을 때, 위상정렬로 가능한 순서와 임계경로는?

→ 위상정렬로 "검사 끝나야 수술" 같은 제약을 지키는 실행 순서를 만들고, 임계경로(최장경로)로 "아무리 빨라도 이만큼은 걸린다"는 하한을 계산합니다.

Q2. 수술실 3개·의료진 한정 시, 최적화 알고리즘으로 총 대기시간을 최소화하면 단순 순서 대비 얼마나 단축?

→ 단순 순서(베이스라인)를 기준으로, RCPSP·GA·SA가 같은 자원 안에서 대기시간을 얼마나 줄이는지 측정합니다. 실측 결과 평균 약 50% 단축.

① 위상정렬 (Topological Sort)

교안 알고리즘 · 모든 스케줄링의 출발점

교안

위상정렬

✅ 적용

화살표(먼저 → 나중)로 이어진 작업들을 순서가 꼬이지 않게 "한 줄"로 세우는 것.

💡 쉽게 말하면

옷 입는 순서를 생각하세요. 양말을 신어야 신발을 신고, 속옷을 입어야 바지를 입습니다. 이런 "먼저 해야 하는 것"을 모두 지키면서 전체 순서를 정하는 게 위상정렬입니다.

🏥 우리 병원 문제에선

환자는 보통 [검사 → 마취 준비 → 수술 → 회복] 순서를 따릅니다. 검사가 안 끝났는데 수술을 시작할 순 없죠. 위상정렬이 이 모든 선후 제약을 지키는 실행 순서를 만들어줍니다.

⚙️ 코드로는 어떻게

graph.py · topological_order() — 들어오는 화살표(선행 작업)가 없는 작업부터 꺼내고, 그 작업을 지우면 새로 선행이 사라진 작업을 또 꺼내는 식으로 반복합니다. NetworkX의 lexicographical_topological_sort를 써서 같은 입력엔 항상 같은 순서가 나오게(재현 가능) 했습니다.

👁️ 화면·결과에선

이 순서가 베이스라인·GA가 일정을 짜기 시작하는 "우선순위 명단"이 됩니다. 웹의 DAG 그래프 탭에서 작업들이 왼쪽→오른쪽 단계로 늘어선 모습이 바로 위상순서의 시각화입니다.

📐 미니 예시
작업 관계: A→B, A→C, B→D, C→D 가능한 위상순서: A → B → C → D ▸ A는 선행이 없으니 맨 앞, D는 B·C 둘 다 끝나야 하니 맨 뒤 ▸ B와 C는 서로 무관 → 순서가 바뀌어도 OK (A,C,B,D 도 가능)

② DAG 최장경로 · CPM (임계경로법)

교안 알고리즘(최장경로) + 최신 기법(CPM 해석) · 비교의 "기준점"

교안최신

DAG 최장경로 / CPM

✅ 적용

작업 사슬 중 "가장 오래 걸리는 길"의 총 길이. 이걸 줄이지 않으면 전체를 더 빨리 끝낼 수 없습니다.

💡 쉽게 말하면

여러 요리를 동시에 만들어도, 가장 오래 걸리는 요리(예: 3시간 스튜)가 끝나야 식사할 수 있습니다. 그 3시간이 바로 임계경로 — 나머지를 아무리 빨리 해도 전체 시간은 이 사슬이 결정합니다.

🏥 우리 병원 문제에선

수술실·의료진이 무한히 많아도 선후관계 때문에 어쩔 수 없이 걸리는 최소 시간입니다. = 이론적 하한. 어떤 똑똑한 알고리즘도 이보다 빨리 끝낼 수는 없습니다.

⚙️ 코드로는 어떻게

graph.py · critical_path() — 위상순서대로 각 작업까지 도달하는 "가장 긴 누적 시간"을 동적계획(DP)으로 계산하고, 그 최댓값과 경로를 돌려줍니다. (CPM = 이 최장경로를 일정 관점으로 해석한 것)

👁️ 화면·결과에선

요약 카드의 "임계경로 길이(이론적 하한)", 그리고 DAG 탭에서 주황색으로 강조된 경로가 그것입니다. 네 알고리즘(베이스라인·RCPSP·GA·SA)의 완료시각(makespan)을 이 하한과 비교해 "최적에 얼마나 가까운지" 가늠합니다.

📐 미니 예시 (괄호=소요시간)
경로 1: A(10) → B(20) → D(5) 합계 = 35 경로 2: A(10) → C(40) → D(5) 합계 = 55 ← 임계경로 ▸ 임계경로 = 55분. C(40)을 줄이지 않으면 전체를 55분보다 빨리 못 끝냄

③ 강연결요소 (SCC)

교안 알고리즘 · 입력 검증 안전장치

교안

강연결요소 (Strongly Connected Components)

✅ 적용

작업들이 서로 빙빙 도는(순환) 모순이 있는지 탐지합니다.

💡 쉽게 말하면

"A는 B가 끝나야 시작, B는 A가 끝나야 시작" 같은 상황은 닭이 먼저냐 달걀이 먼저냐처럼 영원히 시작할 수 없습니다. SCC는 이런 순환 고리를 찾아냅니다.

🏥 우리 병원 문제에선

선후관계가 잘못 입력돼 순환이 생기면 일정 자체가 불가능합니다. SCC는 스케줄링을 시작하기 전에 이를 걸러내는 안전장치 역할을 합니다. (정상 입력은 순환 없는 DAG)

⚙️ 코드로는 어떻게

graph.py · detect_cycles() — NetworkX의 strongly_connected_components로 크기가 2 이상인 묶음(=순환)이 있는지 검사합니다.

👁️ 화면·결과에선

화면에 직접 보이진 않지만, 합성 데이터 생성·위상정렬 전에 "유효한 DAG인지"를 보장해 뒤 단계가 안전하게 돌아가도록 받쳐줍니다.

④ RCPSP (자원제약 스케줄링)

최신 기법 · OR-Tools CP-SAT · 비교의 주역

최신

RCPSP — 자원제약 프로젝트 스케줄링

✅ 적용

한정된 자원 안에서 "수학적으로 최적인" 시간표를 컴퓨터(솔버)가 직접 찾아줍니다.

💡 쉽게 말하면

회의실이 3개뿐인데 회의가 10개라면? 겹치지 않게, 그리고 모두 최대한 빨리 끝나도록 짜는 최적 시간표 문제입니다. 사람이 직관으로 짜는 게 아니라 솔버가 모든 경우를 따져 최적을 찾습니다.

🏥 우리 병원 문제에선

수술실 3개·의료진 5명이라는 한계 안에서 모든 수술을 배치하되, 환자들의 총 대기시간을 최소로 만드는 "정답에 가까운" 일정을 찾습니다.

⚙️ 코드로는 어떻게

rcpsp.py · schedule_rcpsp() (OR-Tools CP-SAT). 각 작업에 시작시간 변수를 두고, 선후관계(end≤start)와 자원용량(AddCumulative)을 제약으로, Σ대기시간을 목적(Minimize)으로 둡니다. 베이스라인 해를 소프트 힌트(AddHint)로 줘서 탐색 초반에 실행가능한 incumbent를 확보합니다. 이는 하드 제약이 아니라 검색 힌트이며, 실제로는 사실상(in practice) baseline보다 나쁜 해가 반환되지 않습니다.

👁️ 화면·결과에선

비교에서 가장 좋은(또는 근접 최적) 결과를 냅니다. 2D 동선·간트에서 RCPSP를 고르면 수술실이 빈틈없이 촘촘하게 채워진 배치가 보입니다. 실측 평균 ~50% 이상 단축.

📐 핵심 아이디어
변수: 각 작업의 시작시간 start[t] 제약: ① 선행 끝나야 시작 ② 동시에 쓰는 수술실 ≤ 3, 의료진 ≤ 5 목적: minimize Σ ( start[t] − ready[t] ) ← 총 대기시간 최소화 ▸ 솔버가 이 조건을 만족하는 수많은 시간표 중 대기 합이 가장 작은 것을 탐색

⑤ 유전 알고리즘 (GA)

최신 기법 · DEAP · 메타휴리스틱(견고성·속도)

최신

유전 알고리즘 (Genetic Algorithm)

✅ 적용

자연의 진화(자연선택)를 흉내 내어 좋은 일정을 점점 찾아갑니다.

💡 쉽게 말하면

품종 개량과 같습니다. 여러 후보 일정을 만들어 두고, 성적이 좋은 것끼리 섞고(교차) 가끔 살짝 바꿔가며(돌연변이) 세대를 거듭하면, 점점 더 나은 일정이 나옵니다.

🏥 우리 병원 문제에선

작업 우선순위 "순서"를 하나의 유전자(순열)로 봅니다. 총 대기시간이 적을수록 적합도(점수)가 높고, 세대를 반복하며 대기시간이 작은 순서를 탐색합니다.

⚙️ 코드로는 어떻게

ga.py · schedule_ga() (DEAP). 순열 인코딩 → 디코더로 실제 일정 생성 → Σ대기시간으로 적합도 측정 → 우수 개체 선택·OX 교차·셔플 돌연변이 → 반복. 적합도 계산은 metrics.evaluate()를 그대로 재사용해 RCPSP와 같은 잣대로 평가합니다.

👁️ 화면·결과에선

RCPSP에 견줄 만한 해를 빠르게 찾습니다. 같은 시드면 결과가 재현됩니다. 2D 동선에서 GA를 골라 RCPSP와 배치·동선이 어떻게 다른지 비교해 볼 수 있습니다.

📐 진화 한 사이클
[후보 일정 N개] → 적합도(=총 대기) 평가 → 좋은 것 선택 → 교차(두 순서를 섞음) → 돌연변이(일부 순서 뒤섞음) → [다음 세대] … 반복 → 가장 대기 적은 순서 채택

⑥ 시뮬레이티드 어닐링 (SA)

최신 기법 · 궤적 기반 메타휴리스틱 · GA와 대비되는 계열

최신

시뮬레이티드 어닐링 (Simulated Annealing)

✅ 적용

금속 담금질을 모사: 단일 해를 조금씩 바꾸되, "온도"가 높을수록 더 나쁜 해도 받아들여 지역 최적을 탈출한다.

💡 쉽게 말하면

지도를 내려놓고 걸으면서 가끔 일부러 언덕을 올라가는 것과 같습니다. 처음엔 멀리 돌아다니다가(탐험), 갈수록 좋은 길만 고르게 됩니다(수렴). 이 "일부러 나쁜 길을 선택하는 확률"이 온도에 비례합니다.

🏥 우리 병원 문제에선

작업 우선순위 순열을 현재 해로 두고, 두 작업의 순서를 교환하는 이웃 이동을 반복합니다. 총 대기시간이 줄어드는 교환은 무조건 수용하고, 악화되는 교환도 exp(−Δ/T) 확률로 수용해 지역 최적 탈출을 시도합니다.

⚙️ 코드로는 어떻게

sa.py · schedule_sa() — 위상순서를 초기해로, 기하적 냉각(α=0.9975)으로 반복. GA·baseline과 동일한 공유 디코더 greedy_resource_schedule()를 써서 자원제약·turnover를 동일하게 강제합니다. 적합도 측정도 metrics.evaluate()로 동일.

👁️ 화면·결과에선

GA(군집 기반)와 대비되는 궤적 기반 메타휴리스틱. 2D 동선·간트에서 SA를 골라 GA·RCPSP와 배치를 비교할 수 있습니다. 같은 시드면 결과 재현. 메타휴리스틱 내부 비교(GA vs SA)까지 가능합니다.

📐 Metropolis 수용 확률
현재 Σwait=200, 이웃 Σwait=203, T=30 P(수용) = exp(−(203−200)/30) = exp(−0.1) ≈ 0.905 ▸ 약 90% 확률로 악화된 이웃도 수용(탈출 시도) ▸ T가 식으면(예: T=1): exp(−3/1) ≈ 0.05 → 거의 수용 안 함(수렴)

⑦ 베이스라인 (단순 그리디)

비교의 출발점 — "최적화 안 한 기본 방식"

기준

베이스라인 (Greedy)

✅ 적용

똑똑한 최적화 없이, 위상순서대로 "지금 비어있는 수술실에 그냥 바로" 배정하는 기본 방식.

💡 쉽게 말하면 / 🏥 병원에선

별다른 고민 없이 순서대로 빈 방에 넣는 현장 직관 수준의 일정입니다. 이걸 기준(0% 개선)으로 삼아, RCPSP·GA가 얼마나 더 나은지를 잽니다.

⚙️ 코드 / 👁️ 화면

baseline.py · greedy_resource_schedule()를 위상순서로 호출합니다. 2D 동선·간트에서 베이스라인을 고르면 최적화가 안 된 헐거운 배치 → 대기시간이 가장 큽니다.

⑧ 공정한 비교 (가장 중요한 설계)

비교가 의미 있으려면 "같은 문제를 같은 잣대로" 풀어야 합니다.

마라톤 기록을 비교하려면 같은 코스·같은 규칙이어야 공정하듯, 네 알고리즘(베이스라인·RCPSP·GA·SA)도 조건이 같아야 합니다.

🎯 동일 목적식 (Σwait)

네 알고리즘 모두 총 대기시간 = Σ(시작−준비완료)를 똑같이 사용합니다. metrics.evaluate()가 유일한 "심판"이고, RCPSP 목적식·GA 적합도·SA 비용함수가 이 식과 글자 그대로 동일합니다.

🔒 동일 자원제약 (수술실 + 의료진)

넷 다 수술실 그리고 의료진 용량을 강제합니다. 베이스라인·GA·SA는 공유 디코더 greedy_resource_schedule()가 시간축 자원량을 검사하고, RCPSP는 AddCumulative로 처리합니다.

🛠️ 실제로 잡은 결함
초기엔 베이스라인·GA가 의료진(staff) 제약을 무시해서 RCPSP만 불리하게 보였음 → 모든 알고리즘이 동일하게 의료진 제약을 지키도록 통일 → 그제서야 공정한 비교가 성립 → 결과: RCPSP·GA 모두 일관되게 ~50% 단축
