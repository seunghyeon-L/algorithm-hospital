🛠️ 왜 OR-Tools를, 그중에서도 왜 CP-SAT를 골랐나

수술 스케줄링(RCPSP)을 풀 도구를 고른 근거 · 고려한 트레이드오프 정리

1왜 OR-Tools인가

OR-Tools = 구글이 만든 무료·오픈소스 "최적화 공구함"입니다. 여러 종류의 최적화 솔버(도구)를 하나의 라이브러리에 묶어 제공해요.

이걸 고른 이유:

  • 무료·오픈소스 — 상용 솔버(Gurobi·CPLEX)는 비싼 라이선스가 필요. 학생 프로젝트에 결정적.
  • 성숙·검증됨 — 산업·연구에서 널리 쓰이는 신뢰받는 라이브러리.
  • 필요한 도구가 다 들어있음 — 특히 우리에게 꼭 필요한 CP-SAT가 포함.
  • 파이썬에서 바로 사용 — pip install ortools 후 모델만 적으면 끝.

2OR-Tools 안에는 어떤 도구들이 있나

OR-Tools (공구함) ├─ CP-SAT ← 우리가 고른 것 (제약 프로그래밍) ├─ 선형해법 MIP/LP (GLOP·SCIP·CBC — 선형 최적화) ├─ 라우팅 솔버 (차량경로·TSP 배달 문제) └─ 그래프 알고리즘 (최대유량·최소비용흐름·매칭)

우리는 이 중 from ortools.sat.python import cp_model 로 CP-SAT 부분만 꺼내 썼습니다.

3우리 문제에 맞는 후보 비교

수술 스케줄링은 RCPSP(자원제약 프로젝트 스케줄링) 문제입니다. 이 모양에 맞는 후보만 추리면:

후보우리 문제 적합성트레이드오프
CP-SAT
(제약 프로그래밍)		 ✅ 선택 스케줄링 전용 부품 내장 + 조합문제에 강함 + 무료 + 빠른 anytime 해.
단, 순수 선형(비용·흐름) 문제엔 MIP보다 약할 수 있음
MIP / 선형해법 △ 가능하나 번거로움 선형·흐름 문제의 챔피언, 최적 증명에 강함.
단 "겹침 금지" 류 제약을 big-M 보조트릭으로 끼워야 해 모델이 지저분·비대해지고, 최강 솔버는 유료
라우팅 솔버 ❌ 부적합 차량 경로·배달(TSP) 전용 — 우리 문제 모양이 아님
그래프 알고리즘 ❌ 부적합 최대유량·매칭 전용 — 스케줄링은 단일 흐름문제가 아님

→ 라우팅·그래프는 문제 모양 자체가 안 맞아 탈락. 진짜 경쟁은 CP-SAT vs MIP였습니다.

4CP-SAT를 고른 결정적 이유 5가지

1
스케줄링 "전용 부품"이 내장돼 있다 ⭐ (가장 큰 이유)
CP-SAT엔 IntervalVar(시간 구간)·AddCumulative(자원 용량)·AddNoOverlap(겹침 금지)가 스케줄링용으로 처음부터 들어있다. "수술실 12개 동시 초과 금지"가 AddCumulative 한 줄로 자연스럽게 표현된다.
↔ MIP라면 "A가 먼저 또는 B가 먼저" 같은 둘 중 하나(disjunction) 제약을 big-M로 끼워넣어야 해 변수가 많아지고 모델이 커진다.
2
우리 문제의 핵심이 "조합 결정(순서)"이다
수술을 어떤 순서로 끼울지가 본질인데, 이런 조합·논리 추론은 CP/SAT(전파 + 충돌학습)가 가장 잘하는 영역이다. MIP는 선형 구조에 강하지 순수 순서 결정엔 상대적으로 약하다.
3
무료·오픈소스 💰
CP-SAT는 공짜. MIP의 최강 솔버(Gurobi·CPLEX)는 비싼 상용 라이선스가 필요하고, 무료 대안(CBC·SCIP)은 더 약하다. 학생 프로젝트엔 결정적.
4
빠른 "지금까지 최선" 해 (anytime)
CP-SAT는 내부 LNS 덕에 좋은 해를 빨리 찾는다. 우리는 15초로 끊어 쓰므로(최적 증명까진 안 감) 이 특성이 딱 맞았다.
5
논문 표준이다
우리가 조사한 수술 스케줄링 논문들(Guo 2026, Ghandehari & Kianfar 2022)도 CP 계열을 표준으로 쓴다. 같은 도구를 써야 비교가 정당해진다.

5정직한 트레이드오프

MIP가 나쁜 게 아니다. 우리 문제도 MIP로 풀 수 있다. 다만 "수술실 겹침 금지" 같은 스케줄링 제약을 표현하기엔 CP-SAT가 훨씬 자연스럽고, 무료이며, anytime 해가 좋아서 골랐을 뿐이다.

한 줄로: MIP = "선형·비용 문제의 챔피언", CP-SAT = "조합·스케줄 문제의 챔피언". 우리 건 후자라 CP-SAT.

6프로젝트 맥락 — 비교 구도에서의 자리

우리 실험 구도는 baseline / CP-SAT / GA / SA였다. 여기서 CP-SAT는 "최적을 지향하는 정확해 대표(gold standard)" 자리로, GA·SA 같은 휴리스틱과 대비시키는 역할이다. 그 자리에 MIP를 넣을 수도 있었지만 — 스케줄링 적합성 + 무료 + 논문 표준 때문에 CP-SAT가 더 나은 선택이었다.
결론 한 장:
① 라우팅·그래프는 문제 모양이 안 맞아 탈락 →
② 진짜 경쟁은 CP-SAT vs MIP →
③ CP-SAT 승리 = 스케줄링 전용 부품 + 조합 강함 + 무료 + anytime + 논문 표준

※ OR-Tools = 구글의 무료 최적화 라이브러리. CP-SAT는 그 안의 한 솔버(제약 프로그래밍). 우리 코드는 엔진을 만든 게 아니라 cp_model을 가져와 "문제(변수·제약·목적)"만 적고 Solve()를 호출한다. · 관련: 상세과정 페이지 ⑤(알고리즘 5종)
